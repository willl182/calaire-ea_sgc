# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

Durante la aplicacion de plantillas se trato inicialmente una conversion automatica desde Markdown/CSV como si fuera equivalente a aplicar la plantilla mediante copia y pegado en Word/Excel. El usuario senalo correctamente que algunos documentos quedaron mal y que el metodo no correspondia a la expectativa inicial.

## User-Requested Corrections

- El usuario rechazo el primer resultado con `NO`.
- Luego aclaro que el problema era haber recreado documentos en vez de copiar/pegar contenido dentro de las plantillas.
- Despues acepto usar Pandoc como metodo pragmatico, pidiendo reaplicar la conversion con plantillas porque el resultado no estaba tan mal.

## The Solution / Workaround

1. Se restauraron los 36 archivos modificados desde el respaldo temporal inicial.
2. Se actualizo el plan para registrar que ese primer intento habia sido descartado.
3. Tras nueva instruccion del usuario, se reaplico la conversion con Pandoc y plantillas de referencia como metodo aceptado.
4. Se validaron integridad Office, encabezados de DOCX y filas de XLSX.
5. Se separaron los commits: primero plantillas/documentos controlados, luego el resto.

## Learnings to Preserve

- No afirmar que Pandoc equivale a copiar/pegar manual en Word; es una conversion basada en estilos y referencia.
- Antes de reemplazar documentos maestros, confirmar si el usuario espera fidelidad visual/manual o conversion automatizada.
- Para tareas documentales SGC, separar commits por alcance funcional: normalizacion de documentos controlados primero, reorganizacion/auxiliares despues.
- Mantener respaldos temporales cuando se hacen reemplazos masivos de Office.
